Penerapan Logika dalam Informatika

Contoh penerapan logika boolean dalam berbagai bidang ilmu komputer

1. Penerapan dalam Pemrograman

Logika boolean secara langsung diterapkan dalam pemrograman melalui struktur kontrol dan ekspresi boolean.

Kondisi If-Else

// Contoh dalam JavaScript let nilai = 85; let kehadiran = 0.8; // Operator AND (∧) if (nilai >= 70 && kehadiran >= 0.75) { console.log("Lulus"); } else { console.log("Tidak lulus"); } // Operator OR (∨) let hari = "Sabtu"; if (hari === "Sabtu" || hari === "Minggu") { console.log("Hari libur"); } else { console.log("Hari kerja"); } // Operator NOT (¬) let login = false; if (!login) { console.log("Silakan login terlebih dahulu"); }

Loop dengan Kondisi

// Contoh dalam Python counter = 0 max_count = 5 # Loop while dengan kondisi AND while counter < max_count and counter != 3: print(f"Counter: {counter}") counter += 1 # Kondisi dalam for loop for i in range(10): if i % 2 == 0 and i > 0: # Bilangan genap dan lebih dari 0 print(f"{i} adalah bilangan genap positif")

2. Penerapan dalam Sirkuit Digital

Gerbang logika adalah implementasi fisik dari operator logika boolean.

Gerbang Logika Simbol Tabel Kebenaran Fungsi
AND ∧ A ∧ B Output 1 hanya jika semua input 1
OR ∨ A ∨ B Output 1 jika minimal satu input 1
NOT ¬ ¬A Output kebalikan dari input
NAND ¬(A∧B) ¬(A ∧ B) Kebalikan dari AND
NOR ¬(A∨B) ¬(A ∨ B) Kebalikan dari OR
XOR ⊕ A ⊕ B Output 1 jika input berbeda

Contoh Sirkuit Half-Adder

Half-Adder adalah sirkuit yang menjumlahkan dua bit biner:

  • Sum (S) = A ⊕ B (XOR)
  • Carry (C) = A ∧ B (AND)

Digunakan sebagai dasar untuk operasi aritmatika dalam CPU.

3. Penerapan dalam Basis Data

Operator logika digunakan dalam query SQL untuk filtering data.

Query dengan Operator Boolean

-- Mencari mahasiswa dengan nilai A dan kehadiran di atas 80% SELECT * FROM mahasiswa WHERE nilai = 'A' AND kehadiran > 0.8; -- Mencari produk yang habis atau stok kurang dari 10 SELECT * FROM produk WHERE stok = 0 OR stok < 10; -- Mencari pengguna yang bukan admin SELECT * FROM users WHERE NOT is_admin = true; -- Kombinasi kondisi kompleks SELECT * FROM orders WHERE (status = 'completed' AND total > 1000000) OR (status = 'pending' AND payment_method = 'credit_card');

4. Penerapan dalam Kecerdasan Buatan

Sistem Pakar dengan Aturan Logika

Sistem pakar menggunakan aturan IF-THEN yang didasarkan pada logika:

// Aturan untuk diagnosis penyakit IF (demam = true AND batuk = true AND sakit_tenggorokan = true) THEN kemungkinan = "Flu" IF ((demam = true AND ruam = true) OR (nyeri_sendi = true)) THEN kemungkinan = "Demam Berdarah" IF (NOT vaksin_covid AND kontak_positif = true) THEN rekomendasi = "Tes COVID-19"

Decision Tree (Pohon Keputusan)

Algoritma machine learning yang menggunakan logika boolean untuk klasifikasi:

                    Apakah suhu > 38°C?
                    /               \
                  Ya                Tidak
                  /                   \
        Apakah batuk?             Sehat
        /         \
      Ya          Tidak
      /             \
   Flu           Demam Biasa

5. Penerapan dalam Jaringan Komputer

Access Control Lists (ACL)

Aturan firewall menggunakan logika boolean:

# Aturan firewall IF (source_ip = "192.168.1.0/24" AND destination_port = 80 OR destination_port = 443) THEN ALLOW ELSE IF (protocol = "SSH" AND NOT source_ip = "10.0.0.0/8") THEN DENY ELSE THEN ALLOW

6. Penerapan dalam Keamanan Sistem

Aturan Autentikasi

// Sistem autentikasi dua faktor function authenticate(username, password, token) { // Logika autentikasi let valid_username = checkUsername(username); let valid_password = checkPassword(password); let valid_token = checkToken(token); // Harus memenuhi semua kondisi return valid_username && valid_password && valid_token; } // Aturan akses berbasis role function canAccess(resource, user) { let hasRole = user.roles.includes(resource.requiredRole); let hasPermission = user.permissions.includes(resource.permission); let notExpired = !resource.isExpired; return hasRole && hasPermission && notExpired; }

7. Contoh Real-World Application

Sistem Peringatan Banjir

// Pseudocode sistem peringatan banjir let curah_hujan = getRainfall(); // mm/jam let ketinggian_air = getWaterLevel(); // cm let durasi_hujan = getRainDuration(); // jam // Aturan logika untuk peringatan if (curah_hujan > 50 && durasi_hujan > 2) { issueWarning("WASPADA HUJAN LEBAT"); } if (ketinggian_air > 100 || (curah_hujan > 30 && ketinggian_air > 50)) { issueAlert("SIAGA BANJIR"); } if (ketinggian_air > 150 && curah_hujan > 20) { issueEmergencyAlert("BAHAYA BANJIR!"); }
